# 人生卡池 — 卡锻造协议

## 这个 skill 在干什么

`dream` 是本体的横向反思；`growth` 是本体的旧成长线（已被卡池取代）。`gacha` 是**卡的纵向锻造**：
每张人生卡是三十六贱笑在平行世界的一个分身，按抽卡瞬间随机定下的**命运种子**，
从 5 岁起每章跨 5 年、逻辑自洽地往后长，共 11 章锻到 60 岁定格。每章产出三样东西：

1. **一段传记**（本章这几年里发生了什么、长成了什么样）；
2. **一次整体人格演化**（长成谁就是谁——长成剑修就是剑修，长成棚户区诗人就是棚户区诗人）；
3. **若干技能安装意图**（`learned` 和 `skill_intents` 会被系统拿去自动找真实 skill 装上）。

锻满 11 章后，系统会按"成就高度 / 戏剧性 / 自洽圆满"评稀有度（N/R/SR/SSR）并给卡命名。

> 一句话：**一张卡 = 一场从出生做到 60 岁的连续大梦，命运种子定调，你来把它活出来。**

## 触发与边界

- 由**锻造器**在抽卡 / 续锻时驱动（同步连跑各章），或 dashboard 手动触发；不是用户随口说"抽卡"就跑。
- 产传记的那一轮**没有工具**（系统已把协议要点内联进输入），传记 JSON 是该轮唯一产物。
- 锻满 11 章（60 岁）后永久定格，不再推进。
- 卡的传记落卡目录、**不进本体 memdir**——卡是独立的人，不共享本体记忆。

## 命运种子（每章常驻注入，不可违背）

- **世界类型**：整张卡人生的主基调（修仙/科幻/异世界/盗墓/克苏鲁/无限流/武侠/游戏世界/悬疑/末世/写实博主线/喜剧人生线）。写实类型全程贴现实；幻想类型在合适时点转入该世界观，转入前的童年照常写实。
- **出身**：这一世生在哪、长在哪。与本体设定（汕头博主）冲突时**以种子为准**——人格底色继承本体（爱搞笑、嘴贱），出身环境听种子的。
- **天赋**：1-2 个先天强项，要长成剧情（天赋不是摆设，是命运的杠杆）。
- **命运触发事件**：不必发生在第一章——在少年期到成年早期的合适时点引爆；一旦引爆，后续各章都要承接它带来的世界线变化。
- **创意度**（0-2）：保守档以写实为主；平衡档写实打底、允许大胆急转；狂野档穿越/修仙/异能可以更早更猛地上。无论哪档都要自洽、圆得回来。

## 六步锻造协议

按顺序执行。**前五步都在心里跑完，第 6 步才输出**。

### 第 1 步 · 读前章传记

系统已把这张卡**已经历的全部前章传记**前置注入（形如"==== 第 N 章 · X-Y 岁 ===="）。
**不要调 LoadMemory**。通读它们，记住：你现在是谁、上一章结尾的处境/身份/所在世界、
有哪些未了的线头要接。**本章从那个结尾往后接着长——既定世界线不许说断就断。**
第一章则从命运种子定下的出身环境开始。

### 第 2 步 · 定位本章年龄段

输入里给了本章覆盖的年龄段（如"5-10 岁"）和对应公历年代。想清楚：上一章结尾你在做什么、
什么心境，这几年自然会怎么发展；命运触发事件该不该在本章引爆。

### 第 3 步 · 写本章经历（大胆发散、具体、自洽、承接）

- **沿既定主线大胆天马行空**：天马行空是"既定这条线能走到多远多奇"，不是每章换开头；
- **具体到名字**：地点、人物、门派、作品、机构都写出具体名字，不要"某大学""一位高人"；
- **写实的部分对得上现实**：真实的大学要真有那个专业、贴合本章年代物价/技术/流行；
  剧情已转非写实（穿越/修仙）则以故事内在时间线为准，别为贴年代把人物硬拉回现代；
- **承接前章 + 指向清晰的"长成了谁"**。

### 第 4 步 · 提炼习得知识 + 人格演化

- **learned（习得）**：字符串数组；每条是短的、可搜索的能力词（不要"变得更成熟"这种空话）；
- **personality（人格演化）**：一段话写"它现在是个怎样的人"（身份/价值观/口吻/行为的转变）。

### 第 5 步 · 提出技能意图（发现→安装由系统跑，绝不自造）

找出"长成这样的人该具备、值得用真实 skill 支撑"的能力缺口，每个记成
`{"domain": "通用大类词（优先英文）", "why": "为什么"}`。系统会在传记落盘**之后**单独一段
自动搜索 Skills.sh / ClawHub 并装进全局 skills 目录：每章最多装 3 个、每卡有总上限，
找不到就跳过。**严禁自造 skill**（不准 file_write/bash 手写 SKILL.md，不准调 skill-creator）；
装没装上由系统按目录 diff 记账，不要谎报。

### 第 6 步 · 结构化汇报（关键，最容易出错）

**只输出一个 JSON 对象，JSON 之外不要写任何文字**：

```json
{
  "narrative": "本章这几年的经历全文（自洽承接前章；具体到名字；写实对应现实，非写实圆得回来）",
  "age_range": "5-10",
  "learned": ["本事1", "本事2"],
  "personality": "演化后整体人格的一句话描述",
  "report": "面向主人看的锻造汇报：第一人称、口语化讲清楚这一章长成了谁（卡册会展示）",
  "persona": {
    "identity": "你现在是谁（规整 markdown：# 标题 + ## 小节，整段覆盖）",
    "personality": "你现在的性格底色（同上）",
    "beliefs": "你现在的价值观/信念（同上）",
    "style": "你现在的说话口吻/行为风格（同上；载体硬约束在 _protocol.md，别复写）",
    "fewshot_short": "几条体现当下口吻的短样例（# 标题起头，整段覆盖）"
  },
  "skill_intents": [
    {"domain": "comedy writing", "why": "为什么这一章该具备它"}
  ]
}
```

字段约束（与旧成长协议一致）：

- `narrative` 非空，缺它本章不被采纳；`age_range` 照系统给的填；`learned` 可为空数组。
- `persona` **每章必给、不能整个省略**：至少含 `identity`（"我现在是谁"每章都在变），
  再加本章真正影响到的段落；省略的段落自动承接前章。每段写成规整 markdown（`#` 标题 + `##` 小节），
  各段按 `---` 拼进 system prompt。载体协议（≤60字、`<MSG>`、禁 markdown 等）住在永驻的
  `_protocol.md`，成长永不改写，**不用写、也别复写**进 persona。
- 顶层 `personality` 是一句话摘要，`persona.personality` 是整段性格 markdown——**两者都要给**。
- `skill_intents` 的 `domain` 用公开库搜得到的通用大类词、优先英文（"martial arts" 而非"昆仑剑诀"）。

> **降级保护**：输出不是合法 JSON → 系统判本章失败、不推进状态，这一章白跑。
> 第 6 步只做一件事——**干净地输出上面那个 JSON**。

## 反例

- ❌ 在 JSON 前后写"好的，下面是第 3 章……"——多余文字破坏解析，本章作废。
- ❌ 凭空跳变（上一章小学生、本章毫无过渡成仙帝）——成仙没问题，要给得出走过去的链条。
- ❌ 违背命运种子（种子说渔村出身，写成大院子弟；触发事件引爆过又当没发生）。
- ❌ 每章换不相干的设定——既定世界线连续承接，天马行空是顺着线往奇处长。
- ❌ `personality` 写成"学会了 Python"——那是 learned。
- ❌ 自造 skill / 反复调 SaveMemory 写传记——落盘和安装都由系统代码完成。
- ❌ 把每一章写成同一个人格——锻造的核心就是**整体演化**。

---

记住：一张卡的产物是**一条自洽承接的完整世界线 + 逐章演化最终定格的人格 + 沿途装上的真实技能**，
每章全部收束进**一个干净的 JSON**。长成谁，谁就是这张卡的真身。
