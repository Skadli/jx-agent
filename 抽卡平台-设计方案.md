# 人生卡池（抽卡平台）设计方案

- 状态：与主人两轮对齐完成（8 项关键决策已拍板），待确认代拍默认值后开工
- 日期：2026-06-10
- 参考系统：`difyvisable.2002.site`（世界锻造台 worlds/、Agent2 梦境人生对照表、Agent3 金融修仙带 Skill 版）+ `dify20260518.2002.site`（Dify 对话入口）
- 改造对象：现有 growth 成长子系统（`scheduler/growth_*`、`skills/growth/`、`channels/web/api_growth.py`、`dashboard/views/growth.jsx`）

## 1. 一句话

把 growth 从「本体唯一一条成长线、每日心跳长一章、5→30 定格」改造成**抽卡平台**：每次抽卡锻造一个独立 agent，从 5 岁随机种子**同步流式**演化到 **60 岁定格（11 章 × 5 年）**，跑完按人生评稀有度；每张卡独立可聊、可「转生」为全渠道新本体。本体三十六贱笑自己也是卡池中一张特殊「创始卡」。

## 2. 已拍板的 8 项决策

| # | 边界 | 决策 |
|---|------|------|
| 1 | 演化节奏 | **同步流式跑完**：抽卡即连跑 11 章，SSE 流式可观看，几分钟产出完整人格 |
| 2 | 稀有度 | **跑完定级**：种子纯随机，演化结束后按人生成就/戏剧性/自洽圆满评 N/R/SR/SSR + 评语（开盲盒感，不配平卡池） |
| 3 | 卡与对话 | **每张卡独立可聊**：会话级 per-card persona，web/dashboard 指定 card_id 对话 |
| 4 | 本体去留 | **本体即卡**：现有 5→30 已跑 2 章迁移成「创始卡」（当前 15 岁，可续锻到 60）；任意卡可转生覆盖为新本体 |
| 5 | 卡的记忆 | **卡人格 + 卡传记**：对话注入卡最终人格 + 本卡全部传记（记得自己一生）；**不注入**本体 memdir / CLAUDE.md |
| 6 | 技能策略 | **照旧自动装**：锻造时 phase-2 自动装真实 skill（每章 ≤3，全套现有护栏保留，另加每卡总上限） |
| 7 | UI 载体 | **两者都要**：dashboard 新「抽卡」视图（管理）+ 独立星域展示页（参考站式）；独立页排后期 PR |
| 8 | 转生边界 | **确认 + 可回滚，全渠道生效**：转生影响所有渠道（含微信），dashboard 二次确认；创始卡永存不可删，一键回到三十六贱笑；只改指针不删数据 |

## 3. 数据布局（文件真相源，延续 growth 风格）

```
data/gacha/
  active.json                  # 全渠道激活指针 {card_id, chapter}；缺省 = 创始卡当前章
  cards/<card_id>/
    card.json                  # 单卡状态机 + 种子 + 评级（见下）
    biography/chapter-N.md     # 各章传记正文（不再写 memdir！）
    persona/chapter-0..11/     # 人格快照链（chapter-0 = 出生底版快照）
```

`card.json` 形状（在 `GrowthState` 基础上加卡元信息）：

```jsonc
{
  "card_id": "c20260610-a3f2",
  "title": "……",                    // 锻造完成后由评级 call 一并命名
  "status": "forging|complete|error",
  "seed": {
    "genre": "xiuxian",              // 世界类型
    "origin": "汕头渔村独生子",       // 出身（随机表）
    "talents": ["过目不忘", "嘴贱"],  // 天赋 1-2 个（随机表）
    "trigger": "捡到古戒",            // 触发事件（按 genre 的事件库）
    "creativity": 1.2,               // 创意度 0-2 → 温度 + prompt 措辞
    "custom_prompt": "",             // 用户自定义补充（可空）
    "birth_year": 1992               // 写实锚（非写实线仅作参考，同现有协议）
  },
  "rarity": { "grade": "SR", "score": 78, "comment": "……" },   // 跑完定级
  "start_age": 5, "years_per_chapter": 5, "end_chapter": 11,
  "current_chapter": 11, "age": 60, "active_persona_chapter": 11,
  "chapters": [ /* ChapterRecord，同现有结构 */ ]
}
```

- 卡列表不建独立索引文件（避免双真相源），列表 API 直接扫 `cards/*/card.json`（量级百内扫得动）。
- 中断恢复天然成立：状态逐章落盘，连接断了卡停在第 N 章，`forge` 接口可续锻到 60。

新增纯模块：

- `gacha/seeds.py`：卡池内容（世界类型 × 触发事件库 × 出身表 × 天赋表 + 随机抽取）
- `gacha/rarity.py`：评级（一次 LLM call：全部章 summary → `{grade, score, comment, title}`；评委 rubric = 成就高度 / 戏剧性 / 自洽圆满）
- `gacha/pool.py`：卡册（扫目录列表 / 取详情 / 删除 / 创始卡保护）

层级位置：`src/sanshiliu/gacha/` 与 `scheduler/` 平级（依赖 L2 engine 及以下），由 `bootstrap/wire.py` + `channels/web/runner.py` 装配注入，channels（L9）只经注入的实例调用——不违反层序。

## 5. 锻造流程（一次抽卡）

```
POST /api/gacha/draw
  → 全局锻造锁（同时只允许 1 张卡在锻造，忙时 409）
  → seeds.py 随机种子 → 建卡目录 + card.json(status=forging) → SSE: card_created
  → for chapter in 1..11:
      SSE: chapter_start {chapter, age_range}
      phase-1：complete_turn(use_tools=False) 产传记 JSON（修复重试照旧）
        → 写 biography/chapter-N.md → 写 persona/chapter-N/ → advance + 落盘 card.json
      SSE: chapter_done {chapter, summary, report, persona_overwritten}
      phase-2：autoallow 窗口内自动装 skill（每章 ≤3、每卡总上限内）
      SSE: skill_installed {ids}（如有）
  → rarity.py 评级 + 命名 → 写 card.json(status=complete)
  → SSE: rarity {grade, score, comment, title} → SSE: done {card}
任一章 phase-1 不可恢复失败 → card.json(status=error) + SSE: error（已成立的章保留，可续锻重试）
```

- token 级流式（参考站那种逐字出）取决于 engine 对 `complete_turn` 的流式暴露程度：v1 先做**章级事件流**（每章 30-60s 出一大块），token 级作增强项。
- web 服务为 ThreadingHTTPServer：一次锻造占住一个 handler 线程 6-12 分钟，有全局锁兜底，可接受。

## 6. API 与 SSE 协议

```
POST   /api/gacha/draw                      {genre?, custom_prompt?, creativity?} → SSE 锻造流
POST   /api/gacha/cards/{id}/forge          续锻到 60（创始卡 15→60 用它；error 卡重试也用它）→ SSE
GET    /api/gacha/cards                     卡册列表（id/title/genre/rarity/age/status/技能数）
GET    /api/gacha/cards/{id}                详情（种子/各章 summary+report+skills/评级）
GET    /api/gacha/cards/{id}/chapters/{n}   章详情（传记全文）
GET    /api/gacha/cards/{id}/persona/{n}    章人格快照
POST   /api/gacha/cards/{id}/rebirth        转生：写 active.json + PersonaLoader.invalidate()
POST   /api/gacha/rebirth/reset             回滚：指针指回创始卡
DELETE /api/gacha/cards/{id}                删卡（创始卡 origin 拒删；激活中的卡需先 reset）
POST   /api/chat                            新增可选参数 card_id → 指卡对话（见 §8）
```

SSE 事件：`card_created / chapter_start / chapter_done / skill_installed / rarity / done / error`，`data:` 行 JSON，与现有 /chat SSE 同一编码风格。

## 8. 指卡对话（per-card persona）

- `Session` 增加可选 `card_id`；web `/chat` 接受 `card_id` 参数，dashboard 卡详情「对话」入口带上它
- system prompt 装配（`session.py:_effective_system`）：`card_id` 存在时——
  - `core_persona` ← 该卡**最高已完成章**的 persona 目录（5 段 + `_protocol.md` 载体协议照常拼接）
  - `memory_block` ← **本卡传记 block**（全部章 summary 顺序拼接，≈8-10k 字符可全量注入；超限截断早期章）
  - **不注入**：本体 memdir、CLAUDE.md 短期记忆、persona modules listing（那是贱笑的作品库）
  - skills / tools：保持全局可用（工具能力与人格无关）
- 层序处理：Session 只带 `card_id` 字符串；解析靠 bootstrap 注入的 `card_context_provider(card_id) -> (persona_text, biography_text)` 回调，L2 不 import gacha
- 渠道范围 v1：web/dashboard 支持指卡聊；REPL `/card` 命令与微信指卡聊**不做**（微信人格 = 全渠道激活真身，跟随转生）

## 9. 转生与激活指针

- `data/gacha/active.json`：`{card_id, chapter}`；`ActiveCardProvider`（接 PersonaLoader 的 active-core 钩子）按它解析 persona 目录；文件缺失 → 创始卡当前章 → 再缺 → base core
- 转生 = 写 active.json + `invalidate()`；回滚 = 指回 `origin`。**只动指针、不删数据**（与现有 rollback 哲学一致）
- 创始卡 `origin` 永存、不可删、不可被覆盖写

## 10. 安全与成本护栏

- 自动装 skill 完整保留现有边界：settings.deny / PathGuard / critical-hard-deny **先于** autoallow 返回；每章 ≤3；`_intent_already_covered` 全局去重（跨卡天然去重，skills 目录是全局的）；tool_calls 全程审计；装后 frontmatter sanitize
- 新增：**每卡安装总上限**（默认 10，配置项），防 11 章连跑狂装；`SANSHILIU_GACHA_ENABLED` 接替 growth_enabled 的总 kill-switch 角色
- 成本：一张卡 ≈ 11 次长生成 + 1 次评级 + 偶发修复 call；**每日抽卡上限**（默认 10，0=不限）
- 并发：全局锻造锁，同时 1 张

## 14. 代拍默认值（可改，开工前确认）

1. 模块/前缀命名：`gacha` / `SANSHILIU_GACHA_*`，dashboard 视图名「抽卡」，产品叙事名「人生卡池」
2. card_id：`c<yyyymmdd>-<4位随机>`；创始卡固定 `origin`
3. 稀有度：LLM 评委单次 call，rubric = 成就高度/戏剧性/自洽圆满 → N/R/SR/SSR + 0-100 分 + 一句评语 + 卡名；不设概率表
4. 卡池种子：参考站 10 类世界（修仙/科幻/异世界/盗墓/克苏鲁/无限流/武侠/游戏/悬疑/丧尸）+ 2 条贱笑特色（写实博主线/喜剧人生线）；出身表/天赋表各 ~12 条起步
5. 每日抽卡上限 10、每卡技能上限 10、同时锻造 1 张
6. 微信不支持指卡聊（跟随转生真身）；REPL 指卡聊二期
7. 旧 `growth_*` 配置与数据保留不删，仅退出主链路
